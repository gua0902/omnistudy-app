# 🎓 OmniStudy App - 補習班/個人學習管理系統

OmniStudy 是一款專為補習與個人學習設計的現代化、響應式 Web 應用程式 (PWA)。系統整合了每日儀表板、月曆/週視圖日程、待辦清單、學科學術問答、以及仿 Discord 風格的學科挑戰出題系統，並搭配 Supabase 提供即時的雲端資料庫託管與多使用者切換功能。

---

## ✨ 核心特色與功能

### 1. 📊 學習狀態儀表板 (Dashboard)
* **今日概況**：以簡潔高級的白卡片風格展示「今日待辦」、「今日行程」、「今日出題」、「今日筆記」的數量。
* **今日焦點**：直接顯示今日待辦與今日日程焦點，使用者可於儀表板直接勾選完成待辦任務。

### 2. 📅 月曆與「週視圖」切換 (Calendar)
* **靈活視圖**：支援頂端「月視圖」與「週視圖」無縫切換，並伴隨流暢的淡入淡出動畫。
* **日程聯動**：新增日程時，可勾選「同步加入待辦清單」，系統會自動生成同名待辦任務。
* **雙擊快捷鍵**：雙擊月曆/週曆中的日期格子，可快速彈出新增日程表單並帶入該日期。

### 3. 🗳 Discord 風格「動態出題挑戰」 (Quiz Builder)
* **動態增刪選項**：仿照 **Discord 投票建立器 (Discord Poll Builder)**，預設顯示 2 個選項，可點選 `+ 新增選項` 增加至最多 4 個，或隨時點選 `X` 移除。
* **直觀指定答案**：選項左側配有打勾核取圈，點選後會亮起綠色（`✓`）直接指定為正確答案。
* **題目附圖**：支援在題目文字正下方直接上傳題目圖片。

### 4. ❓ 學術問答與解題討論區 (Q&A Thread)
* **學科提問**：支援上傳題目照片，針對不懂的題目向大家提問。
* **解答討論串**：其他使用者可以直接在問題下方發表解答，支援上傳算式/解題步驟照片，打造互助社群。

### 5. 🏷 簡約學科分類與篩選 (Filters)
* **精簡科目**：提供 國文、英文、數學、物理、化學、生物、地科、歷史、地理、公民 等主科標籤。
* **無學科關聯 (none)**：當不關聯任何學科時，卡片與詳細視圖會**完全隱藏學科徽章**（不再顯示 `無` 或 `其他` 標籤），維持介面高級感；同時篩選面板支援過濾「無學科關聯」之內容。

### 6. 🔒 權限保護與多使用者切換
* **獨立作者刪改**：所有內容（筆記、日程、待辦、問答、出題）**僅限該內容的原始作者可以編輯與刪除**；非作者的使用者在詳細資訊視圖中將**完全隱藏「編輯」與「刪除」按鈕**。
* **快速身份切換**：可從頂部導覽列下拉選單即時切換使用者，所有分頁與渲染元件皆會同步刷新加載。

## 🛠 技術棧與架構

* **前端核心**：HTML5, CSS3 (Vanilla CSS, 採用毛玻璃磨砂質感、現代漸變色系與響應式佈局)。
* **前端邏輯**：Vanilla JavaScript (ES6+), [Lucide Icons](https://lucide.dev/) 圖示庫, [Marked.js](https://marked.js.org/) Markdown 渲染器。
* **後端/資料庫**：[Supabase](https://supabase.com/) (PostgreSQL + RLS 安全策略)。
* **PWA 支援**：Service Worker (`sw.js`) 配合 `manifest.json`，支援離線快取緩存與行動端安裝為應用程式。

## 🚀 快速上手與部署

### 1. 資料庫建置 (Supabase)
在您的 Supabase 專案中，前往 SQL Editor 並執行專案根目錄下的 [schema.sql](file:///c:/Antigravity/補習app/schema.sql)。這將會為您建立所需的資料表：
* `users`（使用者資料表）
* `notes`（筆記資料表）
* `calendar_events`（行事曆日程表）
* `todos`（待辦任務表）
* `qa_questions`（問答題目表）
* `qa_solutions`（問答解答表）
* `quiz_questions`（挑戰題目表）
* `quiz_answers`（挑戰作答記錄表）

### 2. 本機設定
1. 將專案下載至本機。
2. 複製或修改 `config.js`，填入您的 Supabase 專案連線資訊：
   ```javascript
   const CONFIG = {
       SUPABASE_URL: "您的_SUPABASE_專案_URL",
       SUPABASE_ANON_KEY: "您的_SUPABASE_ANON_KEY"
   };
   ```

### 3. 啟動網頁
由於瀏覽器安全策略（CORS 與 Service Worker 限制），請將本專案透過伺服器啟動（例如使用 VS Code Live Server，或是 Python 的 `python -m http.server 8000`），即可直接在瀏覽器進行操作。
